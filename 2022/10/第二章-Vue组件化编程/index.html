Vue学习笔记的第二章，介绍了组件化编程的基本使用和原理。

模块与组件、模块化与组件化

模块 组件
理解 向外提供特定功能的js程序，一般就是一个js文件 实现应用中局部功能代码和资源的集合
why js文件很多很复杂 一个界面的功能很复杂
作用 复用js，简化js的编写，提高js运行效率 复用编码、简化项目编码、提高运行效率
模块化/组件化 应用中的js以模块来编写，就是模块化的应用 应用中的功能都是多组件的方式编写的

非单文件组件

：一个文件中包含有n个组件

基本使用

  • 使用组件的三大步骤：
    • 定义组件
    • 注册组件
    • 使用组件
  • 定义组件
    • 使用Vue.extend(options)创建，与Vue(options)类似
    • el不要写：最终所有组件都要经过一个vm的管理，由vm中的el决定服务哪个容器
    • data必须写成函数：避免组件被复用是，数据存在引用关系
    • template中多个元素需要div包裹，单元素不用
  • 注册组件
    • 局部注册：靠new Vue时传入components选项
    • 全局注册：Vue.component('组件名', 组件)
  • 使用组件：<组件名></组件名>

几个注意点

  • 组件命名

    • 单个单词：全小写/首字母大写（与开发者工具相呼应）
    • 多个单词：用 - 连接 / 每个单词的首字母大写（只能在脚手架环境下使用，否则报错）
    • 必须回避HTML中已有的元素名称
    • 可以在定义组件时使用name配置项指定组件在开发者工具中呈现的名字

  • 关于组件标签

    • 写法一：<school></school>
    • 写法二：</school> （只能在脚手架环境下使用，否则后续组件不能渲染）

  • 简写方式

    • 把 Vue.extend(options) 替换成 const school = options

    • 底部还是调用了extend

VueComponent构造函数

  1. school组件本质是一个 VueComponent 的构造函数，是 Vue.extend 生成的

  2. 只需要写<school></school>，Vue 解析时就会创建 school 组件的实例对象，实际执行了：new VueComponent(options)

  3. 每次调用 Vue.extend，返回的都是一个全新的 VueComponent

  4. 关于this指向

    1. 组件配置中: data / methods / watch / computed 的函数中的 this 均是 VueComponent 实例对象
    2. new Vue(options) 配置中：data / methods / watch / computed 的函数中的this均是Vue实例对象

  5. VueComponent 实例对象以后简称 vc

一个重要的内置关系

  • VueComponent.prototype.__proto__ === Vue.prototype
  • why：让组件实例对象vc可以访问到Vue原型上的属性和方法
  • 复习JS原型链知识：
    • 每个函数都有一个prototype，即显示原型
    • 每个实例对象都有一个__proto__，称为隐式原型
    • 对象的隐式原型的值 = 对应构造函数的显示原型的值 （保存的都是地址值，指向原型对象）
