第三章 使用Vue脚手架

发表于 2022-10-24 分类于 前端 ， Vue 本文字数： 5.9k 阅读时长 ≈ 5 分钟
Vue学习笔记的第三章，介绍了Vue脚手架的基本使用，todo-list组件化编码案例，一些组件通信方式（prop, 自定义事件，全局事件总线、消息订阅与发布），过渡和动画、配置代理服务器等。

初始化脚手架

render函数

  • Vue = 核心 + 模板解析器
  • vue.js是完整版的Vue
  • vue.runtime.xxx.js是运行版的Vue，只包含核心功能
  • 下面引用的vue.runtime.xxx.js是残缺版本的，没有模板解析器，vm中的模板不能解析（.vue中的可以），需要使用render
  • 函数接收到的createElement函数去指定内容

文件结构

Vue.config.js配置文件

  • 使用vue inspect > output.js 可以查看到Vue脚手架的默认配置
  • 使用vue.config.js可以对脚手架进行个性化定制

ref与props

ref属性

  • 被用来给元素或子组件注册引用信息（id的替代者）
  • 应用在html标签上获取的是真实DOM元素，应用在组件标签上是组件实例对象
  • 使用方式：
    • 打标识：<h1 v-text="msg" ref="title"></h1>
    • 获取：this.$refs.title

props配置

  • 功能：让组件接收外部传过来的数据
  • 传递数据：<Demo name="xxx"/>
  • 接收数据
    • 方式一：直接接收 props:['name']
    • 方式二：限制类型 props:{ name: String }
    • 方式三：限制类型、限制必要性、指定默认值

ps: props是只读的，Vue底层会监测对props的修改，会警告；如果确实要修改，将props的内容复制一份到data，不要重名，修改和使用data中的数据

mixin混入

  • 功能：可以把多个组件共用的配置提取成一个混入对象（代码复用）
  • 定义

  • 使用

    • 全局：import之后，Vue.mixin(xxx)
    • 局部：在vm中：mixins:[h1,h2]

插件

plugins

  1. 插件 (Plugins) 是一种能为 Vue 添加全局功能的工具代码

  2. 使用插件

    1. 通过全局方法 Vue.use() 使用插件。它需要在你调用 new Vue() 启动应用之前完成（也可以传入一个可选的选项对象）：Vue.use(MyPlugin, { someOption: true })

  3. 编写插件 install方法

scoped样式

  • 问题：不同组件的同名style会冲突，以后import的为准

  • 作用：让样式在局部生效，防止冲突

  • 使用：<style scoped>

  • 原理：给标签添加了个特殊属性（随机生成），以示区别

  • <style lang="less"> 可以指定语法，默认css，使用less需要安装less-loader 可以指定语法，默认css，使用less需要安装less-loader

浏览器的本地存储

  1. 存储内容大小一般为5MB

  2. 通过 Window.sessionStorage 和 Window.localStorage 属性实现

  3. 相关API

    1. xxxStrorage.setItem('key','value')

      把键值对添加到存储中，键名存在则更新对应值

    2. xxxStrorage.getItem('key')

      返回键名对应值

    3. xxxStrorage.removeItem('key')

      把对应键值对从存储中删除

    4. xxxStrorage.clear()

      清空存储中所有数据

  4. 备注

    1. sessionStorage中内容随浏览器关闭而消失

    2. localStorafe存储内容，需要手动清除

    3. getItem获取不到的vlue，显示为null

    4. JSON.parse(null)的结果依旧是null

组件的自定义事件

  1. 组件间通信的方式：子组件B→父组件A

  2. 要在A中给B绑定自定义事件（事件的回调在A中）

  3. 使用方式：

    1. 第一种方式：在父组件中：<Demo @atguigu="test">或 <Demo v-on:atguigu="test"

      若想自定义事件只触发一次，使用once修饰符或$once方法

    2. 第二种方式：在父组件中：

  1. 子组件中，触发自定义事件：this.$emit('atguigu', 数据)

  2. 子组件中，解绑自定义事件：this.$off('atguigu') （一般在beforeDestroy）

  3. 组件上也可以绑定原生DOM事件，需要使用native修饰符

  4. 注意：通过this.$refs.xxx.$on('atguigu', 回调函数)绑定自定义事件时，回调函数要么配置在methods中，要么用箭头函数，否则this指向会出错

全局事件总线

  1. 一种组件间通信的方式：任意组件间通信
  2. 安装全局事件总线
  1. 使用

​ ① A组件接收数据，则在A组件中给$bus绑定自定义事件，回调也在A中（合并只能写成箭头函数）

​ ② B组件提供数据：this.$bus.$emit('xxx', 数据)

  1. 最好在beforeDestroy钩子中，用$off解绑当前组件中用到的事件

todo list应用：把父传子传子的改成全局总线

消息订阅与发布

第三方库提供，不同库语法不同

  1. 一种组件间通信的方式：任意组件间通信
  2. 安装：npm i pubsub-js
  3. 引入：import pubsub from 'pubsub-js'
  4. 接收数据：A组件想接收数据，则在A组件中订阅消息，订阅的回调留在A组件自身
  1. 提供数据：pubsub.publish('xxx', 数据)
  2. 最好在beforeDestroy钩子中，用 pubsub.unsuscribe(pid)去取消订阅

nextTick

  1. 语法：this.$nextTick(回调函数)

  2. 作用：在下一次DOM更新结束后执行其指定的回调

  3. when：在改变数据后，要基于更新后的新DOM进行某些操作时，要在nextTick所指定的回调函数中执行（定时器也可以达到这个效果）

过渡与动画

  1. 作用：在插入、更新或移除DOM元素时，在合适的时候给元素添加样式类名（用transition标签包裹，加上特定类名，类名对应动画/过渡，自己决定）
  2. 图示

  1. 写法

    1. 准备好样式：

      1. 元素进入的样式
        1. v-enter进入的起点
        2. v-enter-active 进入过程中（CSS3动画只用这个）
        3. v-enter-to进入的终点
      2. 元素离开的样式
        1. v-leave离开的起点
        2. v-leave-active 离开过程中（CSS3动画只用这个）
        3. v-leave-to离开的终点

      ps: 进入的起点=离开的终点，进入的终点=离开的起点

    2. 使用 <transition> 包裹要过渡的元素，并配置name属性：(没有name就是v-xx)
